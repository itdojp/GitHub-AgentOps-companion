# Skill: docs-update（ドキュメント整備）

## 目的

読者/利用者が「本文/手順だけで導入できる」状態を担保する。

## 入力（必須）

- 対象ページ（パスまたは公開パス）
- 変更内容（追記/修正/削除）と背景
- 制約（用語、表記、断定禁止、参照方針）
- 検証（リンク/アンカー/ビルド/スクリーンショットの要否）

## 出力（成果物）

- 更新されたドキュメント（章/付録/README 等）
- 検証結果（リンク、ビルド、表示確認の証跡）

## 手順

1. 読者の到達点（何ができるようになるか）を明確化
2. 変更を最小差分で実施（章構成/固定パスを壊さない）
3. 相互参照を更新（章内リンク、関連資産への参照）
4. 品質ゲートを実行（lint/build/link-check）
5. 変更理由と影響範囲を PR に記録

## チェック（品質ゲート）

- [ ] 読者が期待する結論/手順が本文だけで分かる
- [ ] 章参照・内部リンク・アンカーが壊れていない
- [ ] 用語/表記の一貫性が保たれている

## 失敗時の対応

- リンク切れ: パス/拡張子/アンカーの整合を確認し、再現手順を残す
- 断定リスク: 変動要素は一次情報参照へ寄せ、日時/バージョン前提を明記
